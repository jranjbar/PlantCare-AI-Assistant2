# رویش‌بان | PlantCare AI Assistant

ربات تلگرامی و داشبورد مدیریتی هوشمند برای شناسایی، عارضه‌یابی و نگهداری از گیاهان — مبتنی بر Gemini AI.

## ساختار پروژه

```
server.ts                 # نقطه ورود سرور (فقط wiring؛ منطق در src/ است)
src/
  api/                     # روترهای Express (هر فایل یک resource)
  auth/                    # احراز هویت ادمین (هش رمز، JWT)
  database/                # لایه ذخیره‌سازی JSON اتمیک + مسیرها + seed
  admin/                   # فرانت‌اند داشبورد مدیریت (React + Vite + Tailwind)
  telegram/                # کلاینت Bot API، کیبوردها، هندلرها، وب‌هوک
  ai/                      # کلاینت Gemini، promptها، اسکیماها، سرویس‌های AI
  services/                # منطق کسب‌وکار (CRM گیاهان، اشتراک، حافظه، زمان‌بند و ...)
  middleware/               # auth، error handler، rate limiter، logger
  utils/                    # ابزارهای عمومی (env، logger، تاریخ، رسانه)
  types/                    # تمام تایپ‌های مشترک بک‌اند و فرانت‌اند
```

## قابلیت‌های اصلی

- **🩺 دکتر هوشمند گیاه**: تحلیل عکس *و ویدئو* گیاه با Gemini و تشخیص بیماری، سلامت و توصیه مراقبتی.
- **🧠 حافظه هوشمند اختصاصی هر کاربر (AI Memory)**: مکالمات هر کاربر به‌مرور خلاصه و در پاسخ‌های بعدی استفاده می‌شود، بدون آنکه به مدل به‌صورت خام انباشته شود.
- **📋 Plant CRM**: هر گیاه یک رکورد کامل دارد (مرحله رشد، مکان، تاریخچه آبیاری، لاگ سلامت، برچسب).
- **🔔 سیستم اعلان تلگرام**: زمان‌بند پس‌زمینه (`node-cron`) هر چند دقیقه یادآورهای سررسیدشده را در تلگرام ارسال می‌کند.
- **🌟 موتور توصیه‌گر**: بر اساس گیاهان CRM و حافظه‌ی کاربر، توصیه‌های شخصی‌سازی‌شده می‌سازد و هر ۲۴ ساعت تازه‌سازی می‌شود.
- **💎 سیستم اشتراک**: سه پلن (رایگان/کشاورز ویژه/تجاری) با محدودیت ماهانه روی اسکن، طرح کشت، چت و سقف تعداد گیاه؛ تحلیل ویدئو فقط در پلن‌های بالاتر.
- **📊 داشبورد مدیریتی**: ورود امن ادمین (JWT + httpOnly cookie)، مدیریت کاربران/CRM/یادآور/توصیه/حافظه/اشتراک، تنظیمات ربات، و آزمایشگاه تست AI.

## نصب و اجرا

```bash
npm install
cp .env.example .env   # سپس مقادیر را پر کنید
npm run dev            # حالت توسعه (Vite + Express روی یک پورت)
```

متغیرهای ضروری در `.env`: `GEMINI_API_KEY`، `TELEGRAM_TOKEN`، `ADMIN_PASSWORD`، `ADMIN_JWT_SECRET`، `TELEGRAM_WEBHOOK_SECRET`.

## ساخت نسخه Production

```bash
npm run build   # خروجی در dist/ (فرانت + server.cjs)
npm start
```

## فعال‌سازی Webhook تلگرام

پس از بالا آمدن سرور روی یک آدرس عمومی، Webhook را با همان سکرتی که در `.env` گذاشتید ثبت کنید:

```bash
curl -X POST "https://api.telegram.org/bot<TOKEN>/setWebhook" \
  -d "url=https://your-domain.com/api/telegram" \
  -d "secret_token=<TELEGRAM_WEBHOOK_SECRET>"
```

## ورود به داشبورد ادمین

پس از اولین اجرا، با `ADMIN_USERNAME`/`ADMIN_PASSWORD` تنظیم‌شده در `.env` وارد شوید (آدرس ریشه سرور). به‌شدت توصیه می‌شود این مقادیر پیش‌فرض را قبل از انتشار تغییر دهید.

## نکات فنی

- دیتابیس یک فایل JSON اتمیک (`data/db.json`) چندکاربره است؛ برای مقیاس بالاتر می‌توان به Postgres/MongoDB مهاجرت کرد بدون تغییر در لایه‌های بالادست (فقط `src/database` تغییر می‌کند).
- رسانه‌های آپلودی (عکس/ویدئو) در `data/media/<chatId>/` ذخیره می‌شوند و thumbnail ویدئو با `ffmpeg` (اگر نصب باشد) ساخته می‌شود.
- تحلیل ویدئو با ارسال مستقیم فایل (تا ۲۰ مگابایت) به Gemini انجام می‌شود.
